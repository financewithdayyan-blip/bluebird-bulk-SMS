📡 SMS Provider

Messages are sent via POST /2010-04-01/Accounts/{SID}/Messages.json. Credentials are saved in localStorage only.

Configured server-side
Zoom credentials are never stored in the browser. Set these in .env.local (and in Vercel → Project → Environment Variables for production):
ZOOM_ACCOUNT_ID · ZOOM_CLIENT_ID · ZOOM_CLIENT_SECRET · ZOOM_FROM_NUMBER · ZOOM_USER_EMAIL
Sends via POST /v2/phone/sms/messages through this app's /api/send-sms route. Use Test Connection below to verify.

Uses POST /v1/messages on the OpenPhone API.

Demo Mode Active
No real SMS will be sent. The app simulates API calls with randomized success/failure. Use this to test the workflow before wiring up a real provider.
